## Relationship to Research

- `/docs/` is **fed by** `/research/` — stable research syntheses are promoted to docs
- **Separate namespaces** — a topic can exist in both areas without duplication
- **Cross-references work both ways** — docs cite research for background; research links to docs for practical application
- **Manual curation** — docs are not automatically generated; user decides when research is ready to curate

---

## Workflow: Promoting Research to Docs

1. Identify a stable research topic/synthesis in `/research/` that's mature and reusable
2. Decide which Diátaxis type it best fits (tutorial, how-to, explanation, reference)
3. Polish and adapt as needed for a curated audience
4. Create page in appropriate `/docs/` subdirectory
5. Update this index
6. Create cross-references between research and docs pages
7. Log the promotion in `/research/log.md`
